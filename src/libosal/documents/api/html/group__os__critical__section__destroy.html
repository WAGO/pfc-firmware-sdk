void os_critical_section_destroy ( handle_t  section)

Any thread of the process can use the os_critical_section_destroy routine to release the system resources were allocated when the critical section object was initialized. After this os_critical_section_destroy has been called, the critical section object can no longer be used for synchronization.

Parameters:
sectionA pointer to the critical section object. The object must have been previously initialized with the os_critical_section_create routine.
Returns:
