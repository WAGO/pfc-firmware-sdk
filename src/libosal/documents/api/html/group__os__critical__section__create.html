handle_t os_critical_section_create ( void  )

The os_critical_section_create routine initializes a critical section object, before it can be used. A critical section which is no longer required should be cleaned up by calling os_critical_section_destroy.

After a critical section object has been initialized, the threads of the process can specify the object in the os_critical_section_lock and the os_critical_section_unlock routine to provide mutually exclusive access to a shared resource. For similar synchronization between the threads of different processes, use a mutex object.

Returns:
A handle to the critical section object indicates success, NULL indicates failure.
