void os_mutex_lock ( handle_t  mutex)

To lock a critical section a call to the os_mutex_lock routine has to be performed. In normal operation the routine will return success after claiming the mutex lock. If another threads currently owns the mutex, the call to os_mutex_lock blocks.

The current owner of a mutex should call os_mutex_unlock, when a lock is no longer required. This operation must be performed by the owner, not by another thread.

Parameters:
mutexSpecifies the reference to the mutex created by a successful call to os_mutex_create.
