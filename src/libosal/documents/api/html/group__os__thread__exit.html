void os_thread_exit ( uint_t  exitCode)

The os_thread_exit allows a thread to terminate itself, thus ensuring that it will not be run again by the scheduler. When this routine is called explicitly or by returning from the thread execution procedure, the thread terminates. The state of the thread object becomes signaled.

The thread remains in use by the operating system and the reference returned by the os_thread_create routine is still valid. The thread should be deleted by a call to the os_thread_destroy routine, after the thread has exited and is no longer required.
