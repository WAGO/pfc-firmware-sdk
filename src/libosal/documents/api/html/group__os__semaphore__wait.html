void os_semaphore_wait ( handle_t  semaphore)

The os_semaphore_wait routine is used by a consumer thread to wait for an event. The semaphore's state is signaled, if its counter is greater than zero. An event has already occurred in the past. In this case the counter will be decremented and the call will return immediately. Otherwise the current thread will be blocked until there is a os_semaphore_post call.

Parameters:
semaphoreSpecifies the reference to the semaphore returned by a successful call to os_semaphore_create.
