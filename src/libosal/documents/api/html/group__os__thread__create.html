handle_t os_thread_create ( uint_t  stack_size,
uint_t  prio,
os_thread_routine_t  routine,
void *  user 
)

The os_thread_create routine creates a thread to execute within the address space specified for a new thread. The thread execution begins at the routine of type os_thread_routine_t specified by the routineThread parameter. If this execution routine returns, the return value is used to terminate the thread in an implicit call to the os_thread_exit routine. If the start address is invalid when the thread runs, an exception occurs, and the thread terminates.

The thread remains in the system until the thread has terminated and all references to it has been closed. A call to the os_thread_destroy removes a created thread and releases the thread resources.

Parameters:
stack_sizeSpecifies the size of the virtual memory reserved for the thread.
prioSpecifies the priority value. Valid values are 0-255 with 0 having the highest priority
routinePoints to the user-defined routine of type os_thread_routine_t to be executed by the thread. This routine represents the starting address of the thread.
userPoints to user specific data passed to the thread.
Returns:
A reference to the created thread indicates success. NULL indicates failure.
Remarks:
The number of threads can create is limited by the available memory.
