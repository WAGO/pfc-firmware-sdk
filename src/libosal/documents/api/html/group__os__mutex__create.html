handle_t os_mutex_create ( tchar_t *  name)

A call to the os_mutex_create routine initializes a mutex before this object can be used. If the mutex is not longer required, the os_mutex_destroy routine can be used. The os_mutex_create routine are normally called during initialization but may also be called from thread context.

The main functions for using a mutex are os_mutex_lock and os_mutex_unlock to lock and to unlock a critical section.

Parameters:
nameSpecifies the name of the mutex object. If name is NULL, the mutex object is created without a name.
Returns:
A handle to the mutex object indicates success, NULL indicates failure.
